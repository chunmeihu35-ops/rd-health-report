## 一、样式问题

### 问题：上月措施跟进位置
- **错误**：放在 KPI 卡片上方
- **正确**：放在「核心问题定位」上方（在数据分析之后、问题定位之前）
- **逻辑**：先展示数据，再回顾上月措施效果，最后定位本月问题

### 问题：指标建议优化格式
- **错误**：简单列表
- **正确**：表格格式（建议/数据依据/状态）

## 二、数据映射问题

### 核心 KPI 映射

| 模板字段 | MCP 字段 | 注意事项 |
|----------|----------|----------|
| 月度需求交付个数 | chart_1 月度统计 | 需加**日均**（总数/工作日） |
| 人均交付个数 | chart_2 人均交付 | 当月 vs 上月（注意方向） |

| 人均缺陷数 | chart_4 人均缺陷 | 当月 vs 上月（注意方向） |

### 工具效率快照映射

| 模板字段 | MCP 字段 | 计算方式 |
|----------|----------|----------|
| 操作次数 | chart_7 关闭数 | 直接取值 |
| 日均操作 | 计算值 | 总次数 / 工作日天数 |
| 人均操作 | chart_8 团队人均有效操作次数 | **不是** 18.06，**是** 13.12 |
| 操作时长 | chart_7 平均时长 | 当月值 |
| 操作时长基准 | chart_7 上月平均时长 | **是上月值**，不是固定值 |

## 四、日均指标计算

| 月份 | 工作日 | 用途 |
|------|--------|------|
| 1月 | 22天 | 正常月 |
| 2月 | 17天 | 春节假期 |
| 3月 | 22天 | 正常月 |
| 4月 | 21天 | 清明1天 |
| 5月 | 20天 | 劳动节 |

**日均计算**：总数 / 工作日天数
**折算增幅**：(当月日均 - 上月日均) / 上月日均

## 五、数据来源链接规则

**数据来源 ≠ 固定链接**

数据来源应该是**实际调用 MCP 获取数据的视图链接**：

```
调用 MCP get_chart_detail 时:
- project_key: fjzixun_pmo
- view_id: sF5rf1cvg  ← 这个是关键

数据来源链接 = https://project.feishu.cn/{project_key}/multiProjectView/{view_id}?scope=workspaces&node={node_id}
```

**SOP 流程要求**：
1. 调用 MCP 时记录使用的 `view_id` 和 `node_id`
2. 报告和卡片中的数据来源链接**动态生成**，指向实际获取数据的视图
3. 不同项目的 view_id 不同，不能写死

## 七、检查清单

生成报告前必须检查：

- [ ] KPI 数据方向：当月 vs 上月（不要搞反）
- [ ] 工具效率快照：人均操作、操作时长基准是否正确
- [ ] 跨角色协作：字段名是否与模板匹配
- [ ] 需求质量：一次验收通过率 ≠ 按时交付率（字段名不同）
- [ ] 日均指标：月度交付、操作次数需加日均
- [ ] 上月措施跟进：位置在核心问题定位之前
- [ ] 短链接格式：驼峰式，无下划线
