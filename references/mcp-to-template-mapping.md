# MCP 数据 → 报告模板字段映射规范

> 2026-04-03 新增：解决 MCP 取数后数据错位问题

## 背景

之前用截图识别方式，字段名称直接从图片 OCR 获取，与模板一致。
换成 MCP 获取数据后，MCP 返回的字段名 ≠ 模板字段名，必须建立映射关系。

### 需求交付周期增长率

| MCP Chart | MCP quota_titles | 说明 |
|-----------|------------|------|
| chart_4 (7618913420485594068) | 增长率 | 返回的是平均交付周期的精确值（如9.3333），不是增长率百分比！|

**铁律：增长率必须用chart_4的精确值计算**：
- 增长率 = (current_month - prev_month) / prev_month × 100%
- 禁止用chart_3的四舍五入整数计算，会产生大偏差

**教训（2026-04-29）**：用chart_3整数算出+85.7%，实际MCP精确值算出+76.2%，偏差9.5个百分点。

## 数据方向规则

**KPI 对比方向**：当月 vs 上月
- 当月数据 = MCP 返回的最新月份
- 上月数据 = MCP 返回的倒数第二个月份
- 环比 = (当月 - 上月) / 上月 × 100%

**趋势表格方向**：从旧到新
- 显示最近 6 个月
- 第一行 = 最旧月份
- 最后一行 = 当月

## 校验规则

生成报告前必须校验：

1. **月份对齐**：所有表格的月份列必须一致
2. **数据范围**：KPI 用最近 2 个月，趋势用最近 6 个月
3. **字段匹配**：每个填充值必须对应正确的 MCP 字段
4. **数值核对**：填充后检查数值与 MCP 原始数据是否一致

## 使用流程

1. 调用 MCP 获取所有 chart 数据
2. 按映射表提取各字段值
3. 校验数据一致性
4. 填充到模板
5. 生成前复核：逐项对比 MCP 原始数据

## 防错铁律（2026-04-29 新增）

### 铁律 A：当月数据必须排除

MCP 返回数据包含当月（未结束），生成报告时必须排除。
示例：4月生成报告，MCP返回2025-08到2026-04，报告取最近6个月且排除当月 = 2025-10到2026-03。

**校验方法**：报告最后一行的月份必须是分析周期月份（如"3月报告"最后一行必须是2026-03），不能是当月。

### 铁律 B：缺陷密度直接使用，禁止二次计算

MCP 返回的"需求交付缺陷密度"已经是最终值，直接展示即可。
禁止：拿这个值当“缺陷总数”再除以需求数。

**教训（2026-04-29）**：MCP字段"需求交付缺陷密度"被误解为"缺陷总数"，然后自行计算"每需求密度"，导致双重错误。

### 铁律 C：字段名以 MCP quota_titles 为准

映射文件中的 MCP 字段名必须与 MCP 实际返回的 quota_titles 严格一致。
新增项目时，先调用 get_chart_detail 获取真实字段名，再写映射。

**教训（2026-04-29）**：原始映射文件手写时用了猜测的字段名（"按时交付率"“逾期需求数”），与实际MCP返回的quota_titles不匹配。
